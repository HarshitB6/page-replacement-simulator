05 / Learn

Algorithm Reference

FIFO

First In, First Out

Replaces the page that has been in memory the longest. Maintains a queue and evicts from the front.

Time: O(n) Space: O(f)
✓ Best: Sequential access with no reuse
✗ Worst: Cyclic access — suffers Belady's Anomaly
LRU

Least Recently Used

Replaces the page that has not been used for the longest time. Based on the principle of temporal locality.

Time: O(n·f) Space: O(f)
✓ Best: Workloads with strong temporal locality
✗ Worst: Large cyclic scans — thrashes badly
OPT

Optimal (Bélády)

Replaces the page that will not be used for the longest time in the future. Theoretical benchmark — requires future knowledge.

Time: O(n·f) Space: O(f)
✓ Best: Always optimal — minimum fault count
✗ Cannot be implemented in real OS
LFU

Least Frequently Used

Replaces the page with the lowest access frequency. Ties broken by LRU order. Rewards pages accessed many times.

Time: O(n·f) Space: O(f)
✓ Best: Skewed access patterns with hot pages
✗ Worst: Cold-start problem — new popular pages evicted early
⚠ Special Case

Belady's Anomaly

Normally, adding more frames reduces page faults. Belady's Anomaly is the counterintuitive phenomenon where increasing the number of frames actually increases the page fault rate for FIFO. It does NOT affect LRU or Optimal.

Classic example: pages 1 2 3 4 1 2 5 1 2 3 4 5 with FIFO: 3 frames → 9 faults, 4 frames → 10 faults.
